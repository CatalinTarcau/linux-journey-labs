# Linux Journey Labs

> A structured Linux and DevOps learning repository built through hands-on production scenarios, technical documentation, and engineering best practices.

---

## Overview

This repository documents my journey of learning Linux from the ground up while developing the mindset required of a DevOps Engineer and Linux Systems Administrator.

Rather than simply following Linux Journey exercises, every concept is explored through:

- Realistic production-inspired scenarios
- Hands-on laboratory exercises
- Troubleshooting investigations
- Technical documentation
- Interview preparation
- Engineering reflections
- Continuous Git version control

The objective is not only to learn Linux commands, but to understand **when**, **why**, and **how** they are used in real production environments.

---

## Learning Philosophy

Throughout this repository, I approach Linux as if I were joining an engineering team responsible for maintaining production systems.

Each new concept is learned by answering three questions:

1. What problem does this solve?
2. How is it used in production?
3. What engineering mindset should I develop?

This methodology emphasizes practical understanding over memorization.
