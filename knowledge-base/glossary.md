# Linux & DevOps Glossary
 
> This glossary contains the most important Linux and DevOps terminology encountered throughout this repository. New terms are added continuously as additional topics are covered.
 
---
 
# Purpose
 
Linux and DevOps introduce a large amount of technical vocabulary.
 
Rather than searching through multiple documents, this glossary provides a centralized reference for important concepts encountered during the learning journey.
 
Each definition focuses on practical understanding rather than academic terminology.
 
---
 
# A
 
## Absolute Path
 
A filesystem path that starts from the root directory (`/`) and uniquely identifies a file or directory regardless of the current working directory.
 
Example:
 
```text
/etc/ssh/sshd_config
```
 
---
 
# B
 
## Bash
 
The Bourne Again Shell.
 
Bash is both a command-line interpreter and a scripting language used to interact with Linux systems and automate administrative tasks.
 
---
 
## Binary
 
An executable program compiled into machine-readable instructions.
 
Most Linux commands such as `ls`, `cp`, and `grep` are binaries.
 
---
 
# C
 
## Configuration File
 
A file containing settings used by an application or service.
 
Most Linux configuration files are stored under:
 
```text
/etc
```
 
---
 
## Command Line Interface (CLI)
 
A text-based interface used to interact with the operating system by entering commands.
 
---
 
# D
 
## Daemon
 
A background process that runs continuously, providing services without direct user interaction.
 
Examples:
 
- SSH
- Nginx
- Docker
 
---
 
## Directory
 
A container used to organize files and other directories.
 
Equivalent to a folder in other operating systems.
 
---
 
# E
 
## Environment Variable
 
A variable maintained by the operating system that stores configuration information used by applications and shell sessions.
 
Example:
 
```bash
PATH
HOME
USER
```
 
---
 
## Executable
 
A file that can be run as a program.
 
Usually requires execute permission.
 
---
 
# F
 
## Filesystem
 
The method used by Linux to organize and store files.
 
Every file belongs somewhere beneath the root directory.
 
---
 
## File Permission
 
Rules defining who can read, write, or execute a file.
 
Permissions are assigned to:
 
- Owner
- Group
- Others
 
---
 
# G
 
## grep
 
A command-line utility used to search text for matching patterns.
 
Widely used for:
 
- Log analysis
- Configuration inspection
- Troubleshooting
 
---
 
# H
 
## Hidden File
 
A file whose name begins with a period (`.`).
 
Example:
 
```text
.bashrc
.gitignore
```
 
Hidden files usually store configuration rather than user documents.
 
---
 
# I
 
## Inode
 
A data structure used by the filesystem to store metadata about a file.
 
It does **not** store the filename itself.
 
Instead, it stores information such as:
 
- Ownership
- Permissions
- Size
- Timestamps
- Disk location
 
---
 
# K
 
## Kernel
 
The core component of the Linux operating system.
 
The kernel manages:
 
- Processes
- Memory
- Hardware
- Devices
- Filesystems
 
---
 
# L
 
## Log File
 
A file recording events generated by applications or the operating system.
 
Most system logs are located inside:
 
```text
/var/log
```
 
Logs are often the first source of information during troubleshooting.
 
---
 
# P
 
## Pipe
 
A mechanism used to connect commands.
 
The output of one command becomes the input of another.
 
Example:
 
```bash
history | grep ssh
```
 
---
 
## Process
 
A running instance of a program.
 
Linux systems commonly execute hundreds of processes simultaneously.
 
---
 
# R
 
## Relative Path
 
A filesystem path interpreted from the current working directory rather than the filesystem root.
 
Example:
 
```text
../logs/application.log
```
 
---
 
## Root Directory
 
The highest directory within the Linux filesystem.
 
Represented by:
 
```text
/
```
 
Every file and directory ultimately belongs beneath the root.
 
---
 
# S
 
## Shell
 
A program that interprets user commands and communicates with the operating system.
 
Common Linux shells include:
 
- Bash
- Zsh
- Fish
 
---
 
## Standard Input (stdin)
 
The default input stream provided to a program.
 
Typically supplied from the keyboard or another command.
 
---
 
## Standard Output (stdout)
 
The default output stream produced by a command.
 
Usually displayed on the terminal unless redirected.
 
---
 
## Script
 
A text file containing commands executed sequentially.
 
Scripts automate repetitive administrative tasks.
 
---
 
# U
 
## Unix Philosophy
 
A design philosophy encouraging small tools that perform one task well and can be combined together.
 
Linux inherits many of these principles.
 
---
 
# V
 
## Variable Data
 
Data expected to change during normal system operation.
 
Examples include:
 
- Logs
- Cache
- Databases
- Temporary runtime files
 
Most variable data is stored under:
 
```text
/var
```
 
---
 
# W
 
## Wildcard
 
A special character used to match multiple filenames.
 
Example:
 
```bash
*.log
```
 
Matches every file ending in `.log`.
