# Linux Fundamentals > This document serves as the primary reference for Linux concepts learned throughout this repository. It is continuously expanded as new topics are covered and should be considered the central knowledge base for Linux fundamentals. --- # Purpose Linux is more than an operating system—it is an ecosystem built around simplicity, modularity, and the philosophy of combining small tools to accomplish complex tasks. Understanding Linux fundamentals provides the foundation required for Systems Administration, DevOps Engineering, Cloud Computing, Cybersecurity, and Site Reliability Engineering. --- # Linux Philosophy One of Linux's greatest strengths is its philosophy. Instead of creating one large program capable of performing every task, Linux provides many small tools. Each tool performs one job well. These tools can then be combined together to solve increasingly complex problems. This philosophy encourages: - Simplicity- Reusability- Automation- Flexibility --- # Filesystem Hierarchy Linux organizes data into a single hierarchical filesystem beginning at the root directory. ```/``` Every file and directory ultimately belongs somewhere beneath this root. Unlike Windows, Linux does not organize storage around drive letters. --- # Important Directories ## / The root of the entire filesystem. Every other directory is located beneath this point. --- ## /etc Contains system-wide configuration files. Typical contents: - Service configuration- Network configuration- User configuration- Application configuration Production importance: Configuration is usually the first place investigated when applications behave unexpectedly. --- ## /home Contains users' personal directories. Each user generally has an individual workspace where documents, scripts, and personal configuration files are stored. --- ## /usr Contains most user applications, libraries, documentation, and executable programs. It is one of the largest directories on a Linux system. --- ## /var Stores variable data. Examples include: - Application logs- Cache- Databases- Mail queues- Temporary runtime information One of the most important locations is: ```/var/log``` This directory is commonly inspected during troubleshooting. --- ## /tmp Contains temporary files. Files stored here are generally not intended for permanent storage and may be automatically removed by the operating system. --- ## /opt Typically contains optional or third-party software installed outside the operating system's default packages. --- ## /proc A virtual filesystem providing information about the running system and active processes. Unlike traditional directories, most files inside `/proc` are generated dynamically. --- # Absolute vs Relative Paths ## Absolute Path An absolute path always begins from the root directory. Example: ```text/etc/ssh/sshd_config``` Advantages: - Independent of the current working directory.- More reliable in scripts.- Preferred for automation. --- ## Relative Path A relative path begins from the current working directory. Example: ```text../logs/application.log``` Advantages: - Shorter to type.- Convenient during interactive terminal sessions. --- # Hidden Files Files beginning with a period (`.`) are hidden by default. Examples: ```text.bashrc.git.gitignore.profile``` These files commonly contain configuration rather than user data. --- # Wildcards Wildcards allow multiple files to be matched using patterns. Examples: ```bash*.log``` Matches every log file. ```bashconfig*``` Matches every filename beginning with "config". Wildcards reduce repetitive typing and simplify automation. --- # Command History Linux stores previously executed commands. Benefits include: - Faster workflow- Reduced typing- Fewer human errors- Easier troubleshooting Useful command: ```bashhistory``` --- # Finding Files ## find Purpose: Searches recursively throughout the filesystem. Examples: ```bashfind . -name "*.conf" find . -type f find /etc -name "app.conf"``` Common options: - `-name`- `-iname`- `-type` Production usage: - Locate configuration files.- Search deployment scripts.- Find logs.- Investigate application resources. --- # Text Processing ## grep Purpose: Filters text by matching patterns. Examples: ```bashgrep "ERROR" application.log grep -i "database" application.log grep -r "timeout" .``` Production usage: - Log analysis- Configuration searches- Incident investigations- Monitoring --- # Unix Pipes Pipes connect commands together. Example: ```bashhistory | grep ssh``` The output of one command becomes the input of another. This is one of the most powerful concepts in Linux. --- # Engineering Mindset Learning Linux is not about memorizing commands. Professional engineers learn: - When to use a command.- Why a command exists.- How commands can be combined.- How to investigate problems systematically. The command itself is only a tool. The engineering mindset determines how effectively that tool is used. --- # Best Practices - Prefer understanding over memorization.- Use absolute paths in automation.- Investigate logs before making assumptions.- Filter information instead of reading everything.- Combine small tools into efficient workflows.- Document lessons learned.- Practice commands regularly. --- # Related Documents - grep-cheatsheet.md- troubleshooting.md-
